#define SPI_MAX_INSTANCES   2

The spi driver provides basic functionality for sending and receiving data via SPI interface. All four common SPI modes are supported. Moreover, only master mode and 8 bytes blocks are supported.

API:

  • Initialize SPI driver: spiInitialize
  • Deinitialize SPI driver: spiDeinitialize
  • Transmit data: spiTxRx

void spiDisableBeforeSleep ( )

Disables SPI interfaces.

Calling this function before Sleep() backups the SPIEN bits of both SPI blocks and sets the SPIEN to 0.

Caution: Must be called before spiEnableAfterSleep() can be called successfully!!!

Returns
: n/a

void spiEnableAfterSleep ( )

Enables SPI interfaces.

Calling this function after Sleep() restires the SPIEN bits of both SPI blocks which have been backuped by spiDisableBeforeSleep().

Caution: Must not be called before spiDisableBeforeSleep() has been called!!!

s8 spiTxRx ( const u8 *  txData,
u8 *  rxData,
u16  length 
)

Transfer a buffer of given length.

This function is used to shift out length bytes of outbuf and write latched in data to inbuf.

Note
Due to the nature of SPI inbuf has to have the same size as outbuf, i.e. the number of bytes shifted out equals the number of bytes latched in again.
Parameters
[in]txData,:Buffer of size length to be transmitted.
[out]rxData,:Buffer of size length where received data is written to OR NULL in order to perform a write-only operation.
[in]length,:Number of bytes to be transfered.
Returns
ERR_IO : Error during SPI communication, returned data may be invalid.
ERR_NONE : No error, all length bytes transmitted/received.
